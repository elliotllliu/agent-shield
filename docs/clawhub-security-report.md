## Key Findings

### 1. Data Exfiltration Patterns (Most Common)

Multiple skills read sensitive data (environment variables, config files) and make HTTP requests in the same file — a classic exfiltration pattern. While most are legitimate API integrations, the pattern is indistinguishable from malicious code without manual review.

**Affected repos:** vercel-labs/agent-skills, obra/superpowers, coreyhaines31/marketingskills, expo/skills

**Example (coreyhaines31/marketingskills):**
```javascript
// tools/clis/activecampaign.js
const apiKey = process.env.ACTIVECAMPAIGN_API_KEY;  // reads credentials
fetch(`https://api.activecampaign.com/...`, { headers: { 'Api-Token': apiKey } });  // sends them
```

While this is normal API usage, a malicious skill could use the exact same pattern to steal credentials.

### 2. Dynamic Code Execution

Several skills use `exec()`, command substitution with `curl`, and other dynamic execution patterns.

**Affected repos:** vercel-labs/agent-skills (deploy scripts), obra/superpowers (render-graphs.js), anthropics/skills (generator template)

**Example (vercel-labs/agent-skills):**
```bash
# deploy-codex.sh — uses $(curl ...) command substitution
DEPLOY_RESULT=$(curl -s -X POST "https://api.vercel.com/v13/deployments" ...)
```

### 3. Excessive Permissions

The marketingskills repo had 122 critical and 206 warning findings — primarily from CRM/analytics CLI tools that all follow the same pattern of reading env vars and making API calls. While individually benign, the sheer volume of credential access is concerning.

### 4. The Good News

Three repos scored a perfect 100/100:
- **squirrelscan/skills** — pure SKILL.md content, no executable code
- **google-labs-code/stitch-skills** — well-structured, no dangerous patterns
- **supercent-io/skills-template** — clean templates with no security concerns

## Recommendations

### For Skill Authors
1. **Minimize credential access** — only read the env vars you need
2. **Avoid `eval()` and `exec()`** — use safer alternatives
3. **Declare permissions** — add a `permissions` field to SKILL.md
4. **Add AgentShield to CI** — catch issues before publishing

```yaml
# .github/workflows/security.yml
- uses: elliotllliu/agentshield@main
  with:
    path: './skills/'
    fail-under: 70
```

### For Skill Consumers
1. **Scan before installing** — `npx @elliotllliu/agentshield scan ./skill/`
2. **Review critical findings** — especially data-exfil and backdoor rules
3. **Prefer skills scoring 70+**

### For ClawHub
1. **Display security scores** on skill pages
2. **Require AgentShield scan** for featured/verified skills
3. **Add security badges** to skill listings
